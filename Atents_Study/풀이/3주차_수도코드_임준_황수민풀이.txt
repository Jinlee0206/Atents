문제: 푸드 파이트 대회

한 선수는 제일 왼쪽에 있는 음식부터 오른쪽으로, 다른 선수는 제일 오른쪽에 있는 음식부터 왼쪽으로 
순서대로 먹는 방식으로 진행됩니다. 중앙에는 물을 배치하고, 물을 먼저 먹는 선수가 승리하게 됩니다.

두 선수가 먹는 음식의 종류와 양이 같아야 하며, 음식을 먹는 순서도 같아야 합니다. 또한, 이번 대회부터는 칼로리가 낮은 음식을 먼저 먹을 수 있게 
배치하여 선수들이 음식을 더 잘 먹을 수 있게 하려고 합니다. 이번 대회를 위해 수웅이는 음식을 주문했는데, 대회의 조건을 고려하지 않고 음식을 주문하여 
몇 개의 음식은 대회에 사용하지 못하게 되었습니다.

입출력예)

food		result
[1, 3, 4, 6]	"1223330333221"
[1, 7, 1, 2]	"111303111"

입출력 예 설명)
3가지의 음식이 준비되어 있으며, 칼로리가 적은 순서대로 1번 음식을 3개, 2번 음식을 4개, 3번 음식을 6개 준비했으며, 
물을 편의상 0번 음식이라고 칭한다면, 두 선수는 1번 음식 1개, 2번 음식 2개, 3번 음식 3개씩을 먹게 되므로 음식의 배치는 
"1223330333221"이 됩니다. 따라서 1번 음식 1개는 대회에 사용하지 못합니다.

두 선수는 1번 음식 3개, 3번 음식 1개를 먹게 되므로 음식의 배치는 "111303111"입니다.

2. 수도코드
----------------------------------------------------------------------------------
using System;
using System.Linq;

public string solution(int[] food) {

      string "왼편 음식들"을 저장할 문자열 선언;

      for (n이 1 부터 food의 길이 까지 1개씩 증가)
		for (w가 0 부터 food[n] / 2 까지 하나씩 증가)
			왼편 음식 문자열에 n을 추가;
        
      string "오른편 음식들"을 왼편의 음식들의 역순으로 초기화;
	string 정답을 저장할 answer에 "왼편 음식들" + '0' + "오른편 음식들"로 초기화;   
     
      return answer;

}

4. 풀이

using System;
using System.Linq;

public class Solution {
   public string solution(int[] food)
        {
            string answer = "";
            string left = "";
            

            for (int n = 1; n < food.Length; n++)
            {
                for (int w = 0; w < food[n] / 2; w++)
                {
                    left += n.ToString();

                }
            }
            string right = new string(left.Reverse().ToArray());
            answer = left + '0' + right;
            return answer;
        }
}
